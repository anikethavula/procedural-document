Introduction

Git and GitHub are essential tools for software developers, data analysts, and anyone working with version control. Git is a distributed version control system that tracks changes in code, while GitHub is a cloud-based platform that facilitates collaboration and repository management.

This guide provides step-by-step instructions on setting up and using Git and GitHub for beginners in a professional setting.

Requirements

Before starting, ensure you have the following:

  • A computer with macOS, Windows, or Linux
  • Git installed (Check by running git --version in the terminal or command prompt)
  • A GitHub account (Sign up at GitHub.com)
  • A code editor (such as Visual Studio Code, Sublime Text, or IntelliJ IDEA)

Safety Information

Backup your work

Before pushing changes, review and save important files.

Use branches

Avoid making direct changes to the main branch to prevent conflicts.

Commit regularly

Frequent commits help track changes effectively.

Check permissions

Ensure you have the necessary access rights before making changes to a repository.

Steps to Use Git and GitHub

1

Install and Configure Git

  1. Download and install Git from git-scm.com.
  2. Open the terminal or command prompt and configure Git with your name and email:
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
2

Create a Repository on GitHub

  1. Log in to your GitHub account.
  2. Click on the New button to create a new repository.
  3. Enter a repository name, select visibility (public or private), and click Create repository.
3

Clone the Repository Locally

  1. Copy the repository URL from GitHub.
  2. Open your terminal and navigate to the folder where you want to store the repository.
  3. Run the following command to clone the repository:
git clone https://github.com/your-username/repository-name.git
4

Make Changes and Commit

  1. Navigate into the cloned repository:
cd repository-name
  1. Add the changes to Git:
git add .
  1. Commit the changes with a message:
git commit -m "Your commit message"
5

Push Changes to GitHub

  1. Upload your commits to GitHub:
git push origin main
  1. Refresh your GitHub repository to confirm the changes.
6

Create a Branch (Optional but Recommended)

  1. Create and switch to a new branch:
git checkout -b feature-branch
  1. Make changes, commit them, and push the branch:
git push origin feature-branch
7

Create a Pull Request

  1. Navigate to your repository on GitHub.
  2. Click on Pull Requests and select New pull request.
  3. Compare changes and submit the pull request for review.
8

Merge Changes and Delete the Branch

  1. Once the pull request is approved, merge it into the main branch.
  2. Optionally, delete the branch after merging to keep the repository clean.
git branch -d feature-branch
git push origin --delete feature-branch

Conclusion and Further Resources
